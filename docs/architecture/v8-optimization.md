# V8 엔진 최적화 전략 ⭐️

DSM은 단순한 상태 관리 라이브러리가 아닙니다. 자바스크립트 엔진(V8)의 동작 원리, 특히 **JIT(Just-In-Time) 컴파일러와 가비지 컬렉터(GC)를 굴복시킬 수준의 최적화**를 코어에 담았습니다.

## 1. Lazy-Proxying & WeakMap 캐싱

렌더링 파이프라인 내에서 매번 새로운 `Proxy`를 동적으로 생성하는 것은 GC에 막대한 부하를 주어 프레임 드랍을 야기합니다. DSM은 이를 방지하기 위해 정교한 **지연 프록싱(Lazy-proxying)**을 수행합니다.

* **WeakMap 도입:** 객체가 이미 래핑된 적이 있다면 `WeakMap`에 캐시된 Proxy 인스턴스를 즉각 반환합니다.
* **GC 친화적 설계:** 원본 객체가 메모리에서 해제될 때 프록시도 함께 `WeakMap`에서 증발하여 메모리 누수 걱정이 없습니다.

## 2. Reflect API의 전면 도입

트랩 내부의 연산을 `target[prop]`으로 단순 처리하지 않고, `Reflect.get(target, prop, receiver)` API를 적극 도입했습니다. 상속 구조나 `this` 바인딩이 복잡한 도메인 객체에서 발생할 수 있는 치명적인 **컨텍스트 소실(Context Loss) 버그**를 완벽하게 차단합니다.

## 3. 배열 하이재킹(Array Hijacking)과 Delta 계산

얕은 복사로 `isMuting` 플래그를 조작하는 꼼수는 V8 환경에서 용납되지 않습니다. 
_"루프 내에서 절대 객체를 동적으로 생성하거나 변이시키지 말라"_는 수칙을 지키기 위해, V8 내장 배열 메서드의 동작 방식에 맞춘 수학적 Delta 알고리즘을 구현했습니다.

변경된 인덱스 범위만을 정확히 계산하여 `REPLACE` 로그를 도출하므로, 대규모 배열 조작 시에도 성능 저하가 발생하지 않습니다.
